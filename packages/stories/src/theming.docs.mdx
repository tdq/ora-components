# Theming

The Aura design system is built on **Material Design 3 (MD3)** design tokens exposed as CSS custom properties. Every component references these tokens — colours, shapes, spacing, elevation, and typography — so you can customise the entire visual language by overriding a single set of variables. No component-specific CSS overrides are needed.

---

## Overview

All styles in Aura use cascading custom properties scoped under the `--md-sys-*` namespace. Components are implemented without hardcoded colour, radius, or spacing values — they reference tokens such as `var(--md-sys-color-primary)` or `var(--md-sys-shape-medium)`. This architecture guarantees:

- **Global consistency** — changing `--md-sys-color-primary` updates every button, field, label, and panel in one place.
- **Theme-agnostic components** — no component re-rendering is needed when switching between light and dark themes.
- **Minimal CSS footprint** — you only write overrides for the tokens you want to change, not selectors per component.

The complete token set is defined in `packages/ora-components/src/index.css` and loaded automatically when you import `@tdq/ora-components`.

---

## Theme switching

The `ThemeManager` singleton manages theme state, persists the user's choice to `localStorage`, and applies the active theme by setting `data-theme` on the document root.

### Import

```typescript
import { themeManager, Theme } from '@tdq/ora-components';
```

### API

| Method / Property | Signature | Description |
|-------------------|-----------|-------------|
| `setTheme()` | `(theme: Theme): void` | Sets the active theme, persists it to `localStorage`, and applies `data-theme` to `<html>`. |
| `getTheme()` | `(): Theme` | Returns the current theme value (`'light'`, `'dark'`, `'green'`, or `'system'`). |
| `theme$` | `Observable<Theme>` | RxJS observable that emits the current theme value on every change. Subscribe to react to theme switches. |

### Theme type

```typescript
type Theme = 'light' | 'dark' | 'green' | 'system' | (string & {});
```

### Behaviour

- **`'system'`** — the default. Listens to `prefers-color-scheme` media query. When the OS preference changes, the theme updates automatically.
- **`'light'`** — forces light mode regardless of OS preference.
- **`'dark'`** — forces dark mode regardless of OS preference.
- **`'green'`** — an alternate light palette with green/teal accents.
- **Custom strings** — any string value can be used as a theme name (see [Custom theme](#custom-theme)).

`setTheme()` writes the value to `localStorage` under the key `'theme'`, so the user's choice persists across page loads.

### Usage example

```typescript
import { themeManager } from '@tdq/ora-components';

// Switch to dark mode
themeManager.setTheme('dark');

// React to theme changes
themeManager.theme$.subscribe(theme => {
  console.log(`Theme changed to: ${theme}`);
});

// Read current theme
const current = themeManager.getTheme(); // 'dark'
```

### `data-theme` attribute

When the theme changes, `ThemeManager` sets `document.documentElement.setAttribute('data-theme', ...)`. All token overrides are scoped to `[data-theme="dark"]`, `[data-theme="green"]`, etc. The light theme uses the bare `:root` selector.

```css
/* Light (default) */
:root {
  --md-sys-color-primary: #0F52BA;
}

/* Dark */
[data-theme="dark"] {
  --md-sys-color-primary: #60A5FA;
}

/* Green alternate */
[data-theme="green"] {
  --md-sys-color-primary: #166534;
}
```

## Custom theme

Creating a custom theme is as simple as defining a new `data-theme` selector with your token overrides. You can switch to it by calling `themeManager.setTheme('your-theme-name')`.

### Global custom theme (amber brand)

```css
[data-theme="amber"] {
  --md-sys-color-primary: #D97706;
  --md-sys-color-on-primary: #FFFFFF;
  --md-sys-color-primary-container: #FEF3C7;
  --md-sys-color-on-primary-container: #451A03;

  --md-sys-color-secondary: #92400E;
  --md-sys-color-on-secondary: #FFFFFF;
  --md-sys-color-secondary-container: #FFFBEB;
  --md-sys-color-on-secondary-container: #292524;

  --md-sys-color-background: #FFFBEB;
  --md-sys-color-on-background: #292524;
  --md-sys-color-surface: #FFFCF0;
  --md-sys-color-on-surface: #292524;
  --md-sys-color-surface-variant: #FDE68A;
  --md-sys-color-on-surface-variant: #451A03;

  --md-sys-color-outline: #D4A373;
  --md-sys-color-error: #DC2626;
  --md-sys-color-on-error: #FFFFFF;
}
```

```typescript
import { themeManager } from '@tdq/ora-components';

themeManager.setTheme('amber');
```

The `data-theme` attribute on `<html>` will be set to `amber`, and all components will immediately reflect the new palette.

### Scoped theming with class selector

You can also scope a custom theme to a specific element by using a class selector instead of `data-theme`:

```css
.my-custom-section {
  --md-sys-color-primary: #7C3AED;
  --md-sys-color-on-primary: #FFFFFF;
  --md-sys-color-primary-container: #EDE9FE;
  --md-sys-color-on-primary-container: #3B0764;
}
```

```typescript
const section = document.querySelector('.my-custom-section');
section.classList.add('my-custom-section');
```

All Aura components inside `.my-custom-section` will inherit the overridden colours, while components outside remain unaffected.

## Elevation & shape

Override elevation shadows and border radii the same way — by setting the corresponding custom properties at any CSS scope.

### Custom elevation

```css
/* Soften all elevations across the app */
:root {
  --md-sys-elevation-level1: 0 2px 4px 0 rgb(0 0 0 / 0.06);
  --md-sys-elevation-level2: 0 6px 12px -2px rgb(0 0 0 / 0.08);
  --md-sys-elevation-level3: 0 12px 24px -4px rgb(0 0 0 / 0.1);
  --md-sys-elevation-level4: 0 20px 40px -6px rgb(0 0 0 / 0.12);
  --md-sys-elevation-level5: 0 28px 56px -12px rgb(0 0 0 / 0.18);
}

/* Scoped: make a specific panel use no shadows */
.panel-flat {
  --md-sys-elevation-level0: none;
  --md-sys-elevation-level1: none;
  --md-sys-elevation-level2: none;
}
```

### Custom shape

```css
/* Rounded corners across the application */
:root {
  --md-sys-shape-small: 8px;
  --md-sys-shape-medium: 12px;
  --md-sys-shape-large: 16px;
  --md-sys-shape-extra-large: 28px;
}

/* Scoped: sharp corners for a data-dense grid */
.data-grid {
  --md-sys-shape-small: 0px;
  --md-sys-shape-medium: 0px;
  --md-sys-shape-large: 0px;
  --md-sys-shape-extra-large: 0px;
}
```
