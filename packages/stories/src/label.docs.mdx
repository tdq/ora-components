# Label

Labels display read-only text content — headings, field captions, status indicators, or any static or dynamic text that updates over time. The `LabelBuilder` provides a reactive API with three Material Design 3 size presets and full RxJS observable support for live-updating text.

Labels are lightweight `<span>` elements suitable for inline display anywhere in the DOM.

---

## When to use

Use `LabelBuilder` when you need to display read-only text that may update over time:

- **Dynamic status text** — status indicators, progress messages, or live counts whose text changes reactively
- **Section headings** — heading labels within forms, panels, or layouts when you need MD3 typography sizing
- **Field captions** — as an alternative to plain text for labels inside a FormBuilder grid

Use plain text or a `<span>` for completely static text that never changes after render. Use `LabelBuilder` when the text is driven by an observable and updates dynamically.

---

## Usage

```typescript
import { LabelBuilder, LabelSize } from '@tdq/ora-components';
import { BehaviorSubject, of } from 'rxjs';

const status$ = new BehaviorSubject('Processing...');

const label = new LabelBuilder()
  .withCaption(status$)
  .withSize(LabelSize.LARGE)
  .build();

document.body.appendChild(label);

// Later — the label updates automatically:
status$.next('Completed');
```

---

## Builder API

All builder methods return `this` for chaining. Pass RxJS observables into the reactive setters — the component handles subscription lifecycle and automatically cleans up when detached from the DOM.

- **`withCaption`** — `(caption: Observable<string>): this`
  Sets the label text content. Updates reactively when the observable emits.

- **`withSize`** — `(size: LabelSize): this`
  Applies one of the three Material Design 3 type scale presets. Defaults to `LabelSize.MEDIUM`.

- **`withClass`** — `(className: Observable<string>): this`
  Appends additional CSS class names to the label element. Useful for dynamic colour or spacing overrides.

- **`withGlass`** — `(): this`
  Enables the glass-effect variant with frosted background, padding, and rounded corners.

- **`build`** — `(): HTMLElement`
  Constructs and returns the `<span>` element. Subscribe to observables and wire up the DOM. Call once per builder.

---

## LabelSize Enum

| Value | Description |
|-------|-------------|
| `LabelSize.SMALL` | Small type scale (`md-label-small`). Use for captions, metadata, or helper text. |
| `LabelSize.MEDIUM` | Medium type scale (`md-label-medium`). The default. Use for standard labels. |
| `LabelSize.LARGE` | Large type scale (`md-label-large`). Use for headings or prominent text. |

## Styling

All visual properties are driven by MD3 design tokens. Override them on `:root` or a scoped parent to restyle this component without touching source.

| CSS variable | Role |
|---|---|
| `--md-sys-color-on-surface` | Primary text colour |
| `--md-sys-color-on-surface-variant` | Secondary / muted text colour |
| `--md-sys-color-error` | Error text colour |
| `--md-sys-spacing-*` | Padding when glass effect is applied |

See the [Theming guide](../?path=/docs/system-theming--docs) for token reference and custom theme examples.

### Additional notes

- **Typography** is driven by Material Design 3 type scale classes: `md-label-small`, `md-label-medium`, and `md-label-large`.
- **Colour** defaults to `text-on-surface` — customise via `withClass` with a named theme colour.
- **Glass effect** applies `glass-effect`, `px-2`, and `rounded-md` for a frosted-background appearance.
- **Transitions** have `transition-all` so class changes (e.g. size or colour swaps) animate smoothly.
- **Subscription cleanup** is automatic via `registerDestroy` when the element is removed from the DOM.
