# NumberField

Number fields are input components for entering and editing numeric values. The `NumberFieldBuilder` provides a reactive API for constructing number inputs with configurable labels, placeholders, formatting, precision, constraints, prefixes, suffixes, and validation — all driven by RxJS observables for live-updating bindings.

Number fields follow [Material Design 3](https://m3.material.io/components/text-fields) input semantics across two field styles: tonal and outlined.

---

## Usage

```typescript
import { NumberFieldBuilder, NumberFieldStyle } from '@tdq/ora-components';
import { BehaviorSubject, of } from 'rxjs';

const value$ = new BehaviorSubject<number | null>(42);

const field = new NumberFieldBuilder()
  .withLabel(of('Invoice Amount'))
  .withPlaceholder(of('Enter a number...'))
  .withValue(value$)
  .withMinValue(of(0))
  .withMaxValue(of(99999))
  .withStyle(of(NumberFieldStyle.OUTLINED))
  .build();

document.body.appendChild(field);
```

---

## Builder API

All builder methods return `this` for chaining. Pass RxJS observables into the reactive setters — the component handles subscription lifecycle and automatically cleans up when detached from the DOM.

- **`withValue`** — `(value: Subject<number \| null>) => this`
  Sets the reactive value stream. Accepts a `Subject` for two-way binding — emit new values to update the field programmatically. `null` represents an empty value.

- **`withPlaceholder`** — `(placeholder: Observable<string>) => this`
  Sets the placeholder text displayed inside the input when no value is entered.

- **`withEnabled`** — `(enabled: Observable<boolean>) => this`
  Controls the interactive state. When `false` the input is disabled with reduced opacity and no pointer events.

- **`withStyle`** — `(style: Observable<FieldStyle>) => this`
  Applies one of the two Material Design 3 field style variants (`TONAL` or `OUTLINED`). Defaults to `FieldStyle.TONAL` if not set.

- **`withError`** — `(error: Observable<string>) => this`
  Sets the validation error message. When the observable emits a non-empty string, the field displays an error state. An empty string clears the error.

- **`withLabel`** — `(label: Observable<string>) => this`
  Sets the label displayed above the input field.

- **`withClass`** — `(className: Observable<string>) => this`
  Appends additional CSS class names to the root container element. Useful for theme overrides or spacing utilities.

- **`withFormat`** — `(format: Observable<string>) => this`
  Sets the number format. Use `"integer"` to disallow decimal points, or a pattern string like `"0.00"` for fixed decimal formatting.

- **`withPrecision`** — `(precision: Observable<number>) => this`
  Sets the decimal precision for display formatting. For example, `2` formats `123.4` as `123.40`.

- **`withLocale`** — `(locale: Observable<string>) => this`
  Sets the locale for number formatting via `Intl.NumberFormat`. For example, `"de-DE"` uses comma as decimal separator.

- **`withMinValue`** — `(min: Observable<number>) => this`
  Sets the minimum allowed value. Values below this are clamped on blur.

- **`withMaxValue`** — `(max: Observable<number>) => this`
  Sets the maximum allowed value. Values above this are clamped on blur.

- **`withStep`** — `(step: Observable<number>) => this`
  Sets the step increment for keyboard arrow navigation. Defaults to `1`.

- **`withPrefix`** — `(text: Observable<HTMLElement \| string>) => this`
  Sets a non-editable prefix element rendered before the input value. Accepts an `HTMLElement` or a string (e.g. `"$"`).

- **`withSuffix`** — `(text: Observable<HTMLElement \| string>) => this`
  Sets a non-editable suffix element rendered after the input value. Accepts an `HTMLElement` or a string (e.g. `"kg"` or `"%"`).

- **`asGlass`** — `() => this`
  Enables the glass-effect variant with frosted background and blur. Overrides the default style colours with translucent surfaces.

- **`asInlineError`** — `() => this`
  Displays validation errors as a field style change with an error icon in the suffix area, rather than support text below the field.

- **`build`** — `() => HTMLElement`
  Constructs and returns the root element. Subscribe to observables and wire up the DOM. Call once per builder.

## FieldStyle Enum

Re-exported as `NumberFieldStyle` for convenience.

| Value | Description |
|-------|-------------|
| `NumberFieldStyle.TONAL` | Filled background style with a bottom active indicator. Default variant. |
| `NumberFieldStyle.OUTLINED` | Transparent background with an outline border. |

## Styling Notes

- **Number alignment**: Values are right-aligned while placeholder text is left-aligned for readability.
- **Prefix/suffix**: Non-editable elements rendered inside the field wrapper. Prefix appears before the value, suffix after.
- **Disabled state**: Applies `opacity-38` and `cursor-not-allowed` — observable-driven via `withEnabled`.
- **Error state**: When `withError` emits a non-empty string, the field border and active indicator switch to error colour. In `asInlineError` mode, an icon with popover replaces the support text.
- **Glass effect**: Overrides the background and border of each style variant with translucent colours and backdrop blur.
- **Height**: Fixed at `48px` (`h-[48px]`) for consistent alignment in forms.
- **Keyboard navigation**: ArrowUp/Down increment/decrement by step, PageUp/Down by step × 10, Home/End jump to min/max.
