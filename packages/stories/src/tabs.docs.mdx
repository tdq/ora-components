# Tabs

Tabs organise content into separate views, allowing users to switch between them without navigating away. The `TabsBuilder` provides a reactive API for constructing tabbed interfaces with configurable caption, description, individual tabs, and glass-effect styling ŌĆö all driven by RxJS observables.

Tabs follow [Material Design 3](https://m3.material.io/components/tabs) tab semantics with a horizontal navigation bar, active indicator line, and a content area that updates reactively as the user switches tabs.

---

## When to use

Use tabs to organise related content into separate views without navigating to a new page:

- **Detail panels** ŌĆö switch between Overview, Transactions, and Documents views for an account
- **Settings interfaces** ŌĆö group configuration sections (Profile, Security, Notifications) into distinct tabs
- **Multi-step workflows** ŌĆö embed tabs inside a dialog or panel for structured data entry
- **Dashboard sections** ŌĆö toggle between chart views, summary tables, and reports

Avoid using tabs for fewer than two views, when content is unrelated, or when users need to compare content across tabs simultaneously ŌĆö consider a split panel layout instead.

## Usage

```typescript
import { TabsBuilder } from '@tdq/ora-components';
import { LabelBuilder } from '@tdq/ora-components';
import { of } from 'rxjs';

const tabs = new TabsBuilder()
  .withCaption(of('Invoices'))
  .withDescription(of('Manage your invoice tabs'));

tabs.addTab()
  .withCaption(of('Drafts'))
  .withContent(new LabelBuilder().withCaption(of('Draft invoices will appear here')).build());

tabs.addTab()
  .withCaption(of('Pending'))
  .withContent(new LabelBuilder().withCaption(of('Pending invoices will appear here')).build());

tabs.addTab()
  .withCaption(of('Paid'))
  .withContent(new LabelBuilder().withCaption(of('Paid invoices will appear here')).build());

const element = tabs.build();
document.body.appendChild(element);
```

## Builder API

All builder methods on `TabsBuilder` return `this` for chaining. Pass RxJS observables into the reactive setters ŌĆö the component handles subscription lifecycle and automatically cleans up when detached from the DOM.

- **`withCaption`** ŌĆö `(caption: Observable<string>) => this`
  Sets the heading label displayed above the tab navigation.

- **`withDescription`** ŌĆö `(description: Observable<string>) => this`
  Sets the description text displayed below the caption.

- **`asGlass`** ŌĆö `() => this`
  Enables the glass-effect variant with translucent colours on the tab labels and active indicator.

- **`addTab`** ŌĆö `() => TabBuilder`
  Creates and returns a new `TabBuilder` for configuring an individual tab. Chain `withCaption`, `withContent`, and `withVisible` on the returned builder.

- **`withClass`** ŌĆö `(className: Observable<string>) => this`
  Appends additional CSS class names to the root container element.

- **`build`** ŌĆö `() => HTMLElement`
  Constructs and returns the root element. Subscribe to observables and wire up the DOM. Call once per builder.

### TabBuilder

- **`withCaption`** ŌĆö `(caption: Observable<string>) => this`
  Sets the label text displayed on the tab button.

- **`withContent`** ŌĆö `(content: ComponentBuilder) => this`
  Sets the content displayed when the tab is active. Accepts any object implementing `ComponentBuilder`.

- **`withVisible`** ŌĆö `(visible: Observable<boolean>) => this`
  Controls the visibility of the tab. When `false`, the tab button is hidden. If the active tab becomes hidden, focus switches to the first visible tab.

## Keyboard

| Key | Action |
|---|---|
| `Tab` / `Shift+Tab` | Move focus between tab navigation and tab content |
| `Enter` / `Space` | Activate the focused tab |
| `Arrow Left` / `Arrow Right` | Switch to the previous / next tab |
| `Home` | Switch to the first tab |
| `End` | Switch to the last tab |

## Styling

All visual properties are driven by MD3 design tokens. Override them on `:root` or a scoped parent to restyle this component without touching source.

| CSS variable | Role |
|---|---|
| `--md-sys-color-primary` | Active tab indicator and text colour |
| `--md-sys-color-surface` | Tab navigation background |
| `--md-sys-color-on-surface-variant` | Inactive tab text colour |
| `--md-sys-color-outline` | Tab divider |

See the [Theming guide](../?path=/docs/system-theming--docs) for token reference and custom theme examples.

### Additional notes

- **Caption and description**: Displayed above the tab navigation on the left side. Caption uses `text-headline-small` with bold weight; description uses `text-body-medium` in `text-on-surface-variant`.
- **Active tab indicator**: Uses a `border-b-2` bottom border in `text-primary` colour for the active tab.
- **Horizontal scrolling**: The tab navigation bar (`overflow-x-auto`) allows users to scroll horizontally when tabs overflow the container width. Scrollbar is hidden with `no-scrollbar`.
- **Glass effect**: Overrides tab colours for translucent appearance ŌĆö active tab uses `text-gray-900` / `text-white/80`, inactive uses `text-gray-600` / `text-white/60`. Caption and description colours also adapt.
- **Tab visibility**: The `withVisible` observable controls `display: none/block` on the tab button. If the currently active tab is hidden, the component automatically switches to the first visible tab.
- **Content area**: Renders inside a `p-4` padded container beneath the tab navigation. Content is replaced reactively when the active tab changes.
