# Layout

The `LayoutBuilder` provides a flexible, reactive API for constructing horizontal and vertical layouts with slot-based content placement. Each slot can hold a single child component and supports configurable sizing, visibility, and alignment ‚Äî all driven by RxJS observables.

Layouts are the foundational building block for arranging panels, sidebars, headers, and content regions in financial dashboards and data-heavy interfaces.

---

## When to use

Use `LayoutBuilder` when you need to arrange multiple components in a horizontal or vertical stack:

- **Page shells** ‚Äî sidebar, main content, and footer regions arranged in a vertical column
- **Split panels** ‚Äî side-by-side panels with proportional widths using `SlotSize` (e.g. 1/4 sidebar + 3/4 content)
- **Dashboard grids** ‚Äî row-based or column-based arrangements of cards, charts, and data tables
- **Conditional layouts** ‚Äî slots that appear or disappear based on application state via `withVisible`

Use CSS Grid or Flexbox directly for simple one-off arrangements. Use `LayoutBuilder` when slots need reactive visibility, proportional sizing, or integration with other component builders.

---

## Usage

```typescript
import { LayoutBuilder, LayoutGap } from '@tdq/ora-components';
import { of } from 'rxjs';

// Vertical layout with three slots
const layout = new LayoutBuilder()
    .asVertical()
    .withGap(LayoutGap.LARGE);

layout.addSlot().withContent(headerComponent);
layout.addSlot().withContent(contentComponent);
layout.addSlot().withContent(footerComponent);

const element = layout.build();
document.body.appendChild(element);
```

## Builder API

All builder methods return `this` for chaining. Pass RxJS observables into reactive setters ‚Äî the component handles subscription lifecycle and automatically cleans up when detached from the DOM.

### LayoutBuilder

- **`addSlot`** ‚Äî `() => SlotBuilder`
  Creates and returns a new `SlotBuilder` for configuring a child slot. Slots are rendered in the order they are added.

- **`asVertical`** ‚Äî `() => this`
  Sets the layout direction to vertical (column). Children stack top-to-bottom.

- **`asHorizontal`** ‚Äî `() => this`
  Sets the layout direction to horizontal (row). Children flow left-to-right. Default if not set.

- **`withGap`** ‚Äî `(gap: LayoutGap) => this`
  Sets the gap between adjacent slots. Accepts a `LayoutGap` enum value. Default is `LayoutGap.MEDIUM`.

- **`withClass`** ‚Äî `(className: Observable<string>) => this`
  Appends additional CSS class names to the root layout element. Useful for theme overrides or spacing utilities.

- **`withAlignment`** ‚Äî `(alignment: Observable<Alignment>) => this`
  Sets the default alignment for all child slots that do not have their own alignment override. Only applies to horizontal layouts.

- **`withGlass`** ‚Äî `() => this`
  Applies the glass-effect styling (frosted background with blur, padding, and rounded corners) to the layout container.

- **`build`** ‚Äî `() => HTMLElement`
  Constructs and returns the layout `<div>` element with all configured slots. Call once per builder.

### SlotBuilder

Returned by `addSlot()`. Each slot wraps a single child component.

- **`withContent`** ‚Äî `(content: ComponentBuilder) => SlotBuilder`
  Sets the child component displayed inside this slot. Accepts any `ComponentBuilder` instance.

- **`withSize`** ‚Äî `(size: SlotSize) => SlotBuilder`
  Sets the slot width as a fraction of available space using the `SlotSize` enum. For horizontal layouts, defaults to equal distribution (`flex-1`). For vertical layouts, defaults to content-fit.

- **`withVisible`** ‚Äî `(visible: Observable<boolean>) => SlotBuilder`
  Controls slot visibility. When the observable emits `false`, the slot is hidden (`display: none`).

- **`withAlignment`** ‚Äî `(alignment: Observable<Alignment>) => SlotBuilder`
  Overrides the layout-level alignment for this specific slot.

## Enums

### LayoutGap

Defines the gap between adjacent slots.

| Value | CSS Value | Description |
|-------|-----------|-------------|
| `LayoutGap.NONE` | `gap-0` | No gap between slots. |
| `LayoutGap.SMALL` | `gap-1` | 4px gap. |
| `LayoutGap.MEDIUM` | `gap-2` | 8px gap (default). |
| `LayoutGap.LARGE` | `gap-4` | 16px gap. |
| `LayoutGap.EXTRA_LARGE` | `gap-8` | 32px gap. |

### SlotSize

Defines the proportional width of a slot relative to available space.

| Value | CSS Class | Fraction |
|-------|-----------|----------|
| `SlotSize.QUARTER` | `basis-1/4` | 1/4 of available space |
| `SlotSize.THIRD` | `basis-1/3` | 1/3 of available space |
| `SlotSize.HALF` | `basis-1/2` | 1/2 of available space |
| `SlotSize.TWO_THIRDS` | `basis-2/3` | 2/3 of available space |
| `SlotSize.THREE_QUARTERS` | `basis-3/4` | 3/4 of available space |
| `SlotSize.FULL` | `basis-full` | Full available width |
| `SlotSize.FIT` | `flex-none` | Sized to content |

For horizontal layouts, slots without an explicit size use `flex-1` (equal distribution). For vertical layouts, slots default to content height.

### Alignment

Defines the horizontal alignment of content within a slot.

| Value | Description |
|-------|-------------|
| `Alignment.LEFT` | Content aligned to the left (default). |
| `Alignment.RIGHT` | Content aligned to the right. |
| `Alignment.CENTER` | Content centered within the slot. |

## Styling

All visual properties are driven by MD3 design tokens. Override them on `:root` or a scoped parent to restyle this component without touching source.

| CSS variable | Role |
|---|---|
| `--md-sys-spacing-*` | Gap between slots ‚Äî `LayoutGap` enum maps to spacing tokens |
| `--md-sys-color-surface` | Layout container background (when glass effect is applied) |

See the [Theming guide](../?path=/docs/system-theming--docs) for token reference and custom theme examples.

### Additional notes

- **Full width**: The layout container always applies `w-full` for full-width behaviour.
- **Direction**: `asVertical()` applies `flex-col`; `asHorizontal()` (default) applies `flex-row`.
- **Gaps** are implemented with Tailwind `gap-*` utility classes based on the `LayoutGap` enum.
- **Slot sizing**: Horizontal slots without an explicit `SlotSize` receive `flex-1` to distribute space equally. Vertical slots default to content height.
- **Visibility**: Toggling slot visibility uses `display: none` via inline styles, driven by the `withVisible` observable. The component handles cleanup on destroy.
- **Glass effect**: `withGlass()` adds `glass-effect`, `p-4`, and `rounded-large` classes to the layout container for a frosted-glass appearance.
- **Alignment**: Layout-level alignment applies only to horizontal layouts. Slot-level alignment always applies. When both are set, the slot-level alignment takes precedence.
