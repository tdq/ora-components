# Grid

The Grid component is a high-performance tabular data display component for Material Design 3. It supports **virtual scrolling** for large datasets, multi-level grouping, pivot tables, inline editing, row actions, multi-select, column resizing, sorting, and glass-effect styling ‚Äî all driven by RxJS observables for live-updating bindings.

The grid is orchestrated by the `GridBuilder<ITEM>` class, which uses a generic type `ITEM` to ensure type safety across columns, actions, and selection state.

---

## When to use

Use the Grid component when you need to display, analyse, or edit tabular data:

- **Data listing** ‚Äî display invoices, transactions, or account registers with sorting and resizable columns
- **Inline editing** ‚Äî allow users to edit cell values directly with commit/revert semantics
- **Multi-level grouping** ‚Äî organise rows by categories (e.g. department > project > line item)
- **Pivot tables** ‚Äî cross-tabulate data for financial summaries and budget analysis
- **Row selection** ‚Äî multi-select rows with checkboxes for batch operations
- **High-volume data** ‚Äî virtual scrolling handles thousands of rows without performance degradation

For simple lists or non-tabular data, consider a `ListBox` or `MultiSelectList` instead.

## Usage

```typescript
import { GridBuilder, ColumnType } from '@tdq/ora-components';
import { of } from 'rxjs';

interface Invoice {
  id: number;
  client: string;
  amount: number;
  status: string;
}

const data$ = of([
  { id: 1, client: 'Acme Corp', amount: 4500, status: 'Paid' },
  { id: 2, client: 'Beta LLC', amount: 2300, status: 'Pending' },
]);

const grid = new GridBuilder<Invoice>()
  .withItems(data$)
  .withHeight(of(400));

const columns = grid.withColumns();
columns.addNumberColumn('id').withHeader('ID').withWidth('60px');
columns.addTextColumn('client').withHeader('Client').withWidth('1fr');
columns.addMoneyColumn('amount').withHeader('Amount').withWidth('100px');
columns.addTextColumn('status').withHeader('Status').withWidth('100px');

document.body.appendChild(grid.build());
```

## Builder API

All builder methods return `this` for chaining unless they return a sub-builder. Pass RxJS observables into reactive setters ‚Äî the component handles subscription lifecycle and automatically cleans up when detached from the DOM.

### Data & Dimensions

- **`withItems`** ‚Äî `(items: Observable<ITEM[]>) => this`
  Sets the data source for the grid. The grid re-renders when new data is emitted.

- **`withHeight`** ‚Äî `(height: Observable<number>) => this`
  Sets the fixed height of the grid container in pixels. If not called, the grid defaults to `height: 100%` of its parent.

- **`withGrouping`** ‚Äî `(fields$: Observable<(keyof ITEM \| string)[]>) => this`
  Enables multi-level grouping by the provided field names. Toggle groups via the expand/collapse UI.

- **`withSort`** ‚Äî `(field: keyof ITEM \| string, direction: SortDirection) => this`
  Sets the initial sort configuration. Accepts `SortDirection.ASC` or `SortDirection.DESC`.

- **`withPivot`** ‚Äî `(config: PivotConfig) => this`
  Enables pivot table mode for data aggregation and cross-tabulation.

### Configuration

- **`withColumns`** ‚Äî `() => ColumnsBuilder<ITEM>`
  Starts the column definition flow. Returns a `ColumnsBuilder` for adding typed columns.

- **`withToolbar`** ‚Äî `() => ToolbarBuilder`
  Adds an optional top toolbar with primary, secondary, and text buttons. Returns a `ToolbarBuilder`.

- **`withActions`** ‚Äî `() => ActionsBuilder<ITEM>`
  Adds per-row action buttons in a dedicated sticky trailing column. Returns an `ActionsBuilder`.

- **`asMultiSelect`** ‚Äî `() => this`
  Enables row selection via checkboxes with "Select All" in the header.

- **`asEditable`** ‚Äî `(onCommit: (item: ITEM) => void) => this`
  Enables inline cell editing. Press **Enter** to commit an edit, **Escape** to revert.

- **`asGlass`** ‚Äî `() => this`
  Enables translucent glass styling with backdrop blur on the header. Overrides default surface colours.

- **`build`** ‚Äî `() => HTMLElement`
  Constructs and returns the grid container element. Subscribe to observables and wire up the DOM. Call once per builder.

## Column Configuration

### ColumnsBuilder

All column builder methods accept a `dtoField` (the key in the data item) and return a dedicated column builder for that column type:

- **`addTextColumn(field)`** ‚Äî `TextColumnBuilder<ITEM>`
  Standard text display.

- **`addNumberColumn(field)`** ‚Äî `NumberColumnBuilder<ITEM>`
  Numeric data with decimal precision control.

- **`addDateColumn(field)`** ‚Äî `DateColumnBuilder<ITEM>`
  Date display via `toLocaleDateString()`.

- **`addDateTimeColumn(field)`** ‚Äî `DateTimeColumnBuilder<ITEM>`
  Date and time display via `toLocaleString()`.

- **`addEnumColumn(field)`** ‚Äî `EnumColumnBuilder<ITEM>`
  Mapped display values from a caption provider.

- **`addBooleanColumn(field)`** ‚Äî `BooleanColumnBuilder<ITEM>`
  Boolean values rendered as text or checkbox.

- **`addPercentageColumn(field)`** ‚Äî `PercentageColumnBuilder<ITEM>`
  Percentage display (multiplies by 100, appends `%`).

- **`addMoneyColumn(field)`** ‚Äî `MoneyColumnBuilder<ITEM>`
  Currency-formatted numeric data from `Money` objects.

- **`addButtonColumn(field)`** ‚Äî `ButtonColumnBuilder<ITEM>`
  Inline action buttons per row.

- **`addIconColumn(field)`** ‚Äî `IconColumnBuilder<ITEM>`
  Dynamic icon display based on field values.

- **`addCustomColumn()`** ‚Äî `CustomColumnBuilder<ITEM>`
  Fully custom rendering for complex cell content.

### BaseColumnBuilder (Shared Methods)

All column builders inherit these common methods:

- **`withHeader`** ‚Äî `(header: string) => this`
  Sets the display name in the column header.

- **`withWidth`** ‚Äî `(width: string) => this`
  Sets CSS width (e.g. `'100px'`, `'2fr'`, `'15%'`).

- **`withMinWidth`** ‚Äî `(minWidth: string) => this`
  Sets a minimum width for the column.

- **`asSortable`** ‚Äî `(sortable?: boolean) => this`
  Enables sorting UI for the column header. Defaults to `true`.

- **`asResizable`** ‚Äî `(resizable?: boolean) => this`
  Enables column resizing via a drag handle in the header. Defaults to `true`.

- **`asEditable`** ‚Äî `() => this`
  Marks the column as inline-editable. Only effective when the grid has `asEditable()` called.

- **`withAlign`** ‚Äî `(align: 'left' \| 'center' \| 'right') => this`
  Sets text and flex alignment for the column header and cells.

- **`withClass`** ‚Äî `(classProvider: (item: ITEM) => string) => this`
  Adds custom CSS classes to all cells in this column via a provider function.

- **`withSortValue`** ‚Äî `(provider: (item: ITEM) => any) => this`
  Custom sort value provider for the column.

### Specialised Column Methods

- **`NumberColumnBuilder`** ‚Äî `withDecimals(decimals: number): this`
  Sets decimal places (default 2).

- **`MoneyColumnBuilder`** ‚Äî `withCurrencies(currencies: string[]): this`
  Restricts available currencies in the editor. `withPrecision(precision: number): this` ‚Äî sets decimal precision for display and editing.

- **`EnumColumnBuilder`** ‚Äî `withItemCaptionProvider(provider: (item: ITEM) => string): this`
  Custom caption display for each row.

- **`ButtonColumnBuilder`** ‚Äî `withLabel(label: string): this`
  Sets the button text. `withClick(click: ClickListener<ITEM>): this` ‚Äî click handler receiving the row item. `withStyle(style: ButtonStyle): this` ‚Äî button style variant.

- **`IconColumnBuilder`** ‚Äî `withIconProvider(provider: (item: ITEM) => string): this`
  Returns a CSS class string for the icon. `withTooltipProvider(provider: (item: ITEM) => string): this` ‚Äî hover tooltip text.

- **`CustomColumnBuilder`** ‚Äî `withRenderer(renderer: (item: ITEM) => HTMLElement \| string): this`
  Custom render function returning a DOM element or HTML string.

## Row Actions (ActionsBuilder)

When `withActions()` is called, the grid adds a sticky actions column on the right side of every row.

`ActionsBuilder.addAction(icon: string, label: string, onClick: (item: ITEM) => void): ActionBuilder<ITEM>`

The returned `ActionBuilder<ITEM>` provides further configuration:
- `withEnable(enable: (item: ITEM) => boolean): this` ‚Äî conditionally enables the action per row.
- `withVisible(visible: (item: ITEM) => boolean): this` ‚Äî conditionally shows or hides the action per row.

The `icon` parameter must be an SVG string (use `Icons.*` from `@tdq/ora-components`). The `label` is used for the tooltip and aria-label.

---

## Toolbar (ToolbarBuilder)

When `withToolbar()` is called, the grid renders a toolbar above the table.

- **`withPrimaryButton`** ‚Äî `() => ButtonBuilder`
  Adds a filled primary button aligned to the right.

- **`addSecondaryButton`** ‚Äî `() => ButtonBuilder`
  Adds an outlined secondary button.

- **`addTextButton`** ‚Äî `() => ButtonBuilder`
  Adds a text-style button aligned to the left.

- **`withEnabled`** ‚Äî `(enabled: Observable<boolean>) => this`
  Controls the enabled state of all toolbar buttons.

- **`asGlass`** ‚Äî `() => this`
  Applies glass styling to all toolbar buttons.

## Pivot Tables

Pivot mode transforms data into a cross-tabulated layout using row fields, column fields, and aggregated values. Configured via `withPivot(PivotConfig)`.

### PivotConfig

```typescript
interface PivotConfig {
  rows: string[];               // Fields to use as row headers
  columns: string[];            // Fields to use as column headers (pivoted)
  values: PivotValueConfig[];   // Aggregated value fields
  showGrandTotal?: boolean;     // Show grand total column/row
}

interface PivotValueConfig {
  field: string;
  aggregation: AggregationType;  // SUM, COUNT, AVG, MIN, MAX
  header?: string;
}
```

## Keyboard

| Key | Action |
|---|---|
| `Tab` / `Shift+Tab` | Move focus between cells and interactive elements |
| `Enter` | Commit inline edit, expand/collapse group row |
| `Escape` | Revert inline edit, close dropdown editor |
| `Arrow keys` | Navigate between cells (Up, Down, Left, Right) |
| `Space` | Toggle multi-select checkbox on focused row |
| `Page Up` / `Page Down` | Scroll one page up / down |

## Styling

All visual properties are driven by MD3 design tokens. Override them on `:root` or a scoped parent to restyle this component without touching source.

| CSS variable | Role |
|---|---|
| `--md-sys-color-surface` | Grid background |
| `--md-sys-color-surface-container-low` | Header background |
| `--md-sys-color-surface-variant` | Row hover state |
| `--md-sys-color-on-surface-variant` | Header text colour |
| `--md-sys-color-outline` | Column borders and dividers |
| `--md-sys-color-primary` | Selection highlight |
| `--md-sys-shape-large` | Grid border radius |

See the [Theming guide](../?path=/docs/system-theming--docs) for token reference and custom theme examples.

### Additional notes

- **Container**: `bg-background`, `border-outline/30`, `rounded-lg`.
- **Header height**: 52px. The header **row** owns the background (`bg-surface-container-low` in normal mode, `glass-effect !bg-white/20` with backdrop blur in glass mode); header **cells** are transparent and inherit it. The bottom divider is drawn per header cell (`border-b border-outline/20`), not on the row, so it lines up with column geometry. Typography is uppercase, `font-semibold`, `text-[11px]`, `tracking-wider`.
- **Cells**: `px-4`, `flex items-center`. Body cells carry their own bottom border (`border-b border-outline/10 dark:border-stone-50/10`) so borders track columns under sticky/scrolled states.
- **Row height**: 52px with zebra striping on data columns (`bg-surface-container-low/20` for odd rows). Hover state adds `hover:bg-surface-variant/20` and a left accent border.
- **Sticky actions column**: `sticky right-0 z-10`. Uses a single uniform background for all non-selected, non-glass rows ‚Äî no zebra striping in this column. Selected rows use `bg-primary/10`.
- **Glass effect**: Enabled via `asGlass()`. Only the header receives backdrop blur; rows remain transparent.
- **Virtual scrolling**: Uses `requestAnimationFrame` for scroll updates, `transform: translateY` for row positioning, and a buffer of 5 extra rows above and below the viewport.
- **Selection**: Checkbox-based multi-select with header "Select All". Selection updates are optimised to avoid full row re-renders.
