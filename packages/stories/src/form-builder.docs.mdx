# Form

The `FormBuilder` provides a structured, reactive API for constructing data-entry forms with labelled fields, validation, and action toolbars. It combines a fields grid, an optional toolbar, and header (caption + description) into a cohesive layout ã all driven by RxJS observables.

Forms are ideal for accounting, invoicing, and financial data entry flows where structured input with real-time validation is required.

---

## When to use

Use `FormBuilder` when you need a structured data-entry layout with labelled fields and action buttons:

- **Data entry flows** ã structured forms for invoices, payments, account creation, and other financial records
- **Built-in validation** ã field-level and global error observables for real-time validation feedback
- **Reactive layout** ã fields, headings, and toolbars arranged in a CSS grid with column control

Use a manual layout with `LayoutBuilder` and individual field builders when you need full control over spacing, alignment, or non-standard field arrangements that do not fit a grid pattern.

---

## Usage

```typescript
import { FormBuilder, TextFieldStyle } from '@tdq/ora-components';
import { of, BehaviorSubject, combineLatest } from 'rxjs';
import { map } from 'rxjs/operators';

const firstName$ = new BehaviorSubject('');
const lastName$ = new BehaviorSubject('');

const isFormValid$ = combineLatest([firstName$, lastName$]).pipe(
    map(([first, last]) => first.trim() !== '' && last.trim() !== '')
);

const form = new FormBuilder()
    .withCaption(of('Invoice Recipient'))
    .withDescription(of('Enter the recipient details for this invoice.'));

const fields = form.withFields(2);
fields.addTextField(1, 1).withLabel(of('First Name')).withValue(firstName$);
fields.addTextField(2, 1).withLabel(of('Last Name')).withValue(lastName$);

const toolbar = form.withToolbar();
toolbar.withPrimaryButton()
    .withCaption(of('Save Recipient'))
    .withEnabled(isFormValid$);

const element = form.build();
document.body.appendChild(element);
```

## Builder API

All builder methods return `this` for chaining. Pass RxJS observables into reactive setters ã the component handles subscription lifecycle and automatically cleans up when detached from the DOM.

### FormBuilder

- **`withCaption`** ã `(caption: Observable<string>) => this`
  Sets the form heading label rendered above the fields. Pass an observable that emits the caption text.

- **`withDescription`** ã `(description: Observable<string>) => this`
  Sets a description line rendered below the caption. Useful for instructions or context.

- **`withEnabled`** ã `(enabled: Observable<boolean>) => this`
  Controls the enabled state of all child fields and toolbar buttons. When `false`, inputs are disabled.

- **`withError`** ã `(error: Observable<string>) => this`
  Sets a global error message displayed above the toolbar. The message is visible only when the observable emits a non-empty string.

- **`asGlass`** ã `(isGlass?: boolean) => this`
  Enables the glass-effect variant on fields and toolbar. The form container itself is unaffected. Defaults to `true`.

- **`withFields`** ã `(columnsAmount?: number) => FieldsBuilder`
  Creates and returns a `FieldsBuilder` for adding input fields. The optional parameter sets the grid column count (default `1`).

- **`withToolbar`** ã `() => ToolbarBuilder`
  Creates and returns a `ToolbarBuilder` for adding action buttons at the bottom of the form.

- **`build`** ã `() => HTMLElement`
  Constructs and returns the form element with all configured fields, toolbar, and header. Call once per builder.

### FieldsBuilder

Returned by `withFields()`. Use it to add individual field components to a CSS grid layout.

- **`addTextField`** ã `(column?: number, colspan?: number) => TextFieldBuilder`
  Adds a text input field.

- **`addPasswordField`** ã `(column?: number, colspan?: number) => TextFieldBuilder`
  Adds a password input field (masked characters).

- **`addEmailField`** ã `(column?: number, colspan?: number) => TextFieldBuilder`
  Adds an email input field with built-in email validation.

- **`addNumberField`** ã `(column?: number, colspan?: number) => NumberFieldBuilder`
  Adds a numeric input field.

- **`addMoneyField`** ã `(column?: number, colspan?: number) => MoneyFieldBuilder`
  Adds a currency/money input field.

- **`addComboBoxField`** ã `(column?: number, colspan?: number) => ComboBoxBuilder`
  Adds a dropdown / combo-box selector field.

- **`addDatePickerField`** ã `(column?: number, colspan?: number) => DatePickerBuilder`
  Adds a date picker field.

- **`addCheckBox`** ã `(column?: number, colspan?: number) => CheckboxBuilder`
  Adds a checkbox field.

- **`addHeading`** ã `(column?: number, colspan?: number) => LabelBuilder`
  Adds a section heading label within the field grid. Useful for grouping related fields.

- **`withEnabled`** ã `(enabled: Observable<boolean>) => this`
  Propagates enabled state to all child fields. Called automatically by the parent `FormBuilder.build()`.

- **`asGlass`** ã `(isGlass?: boolean) => this`
  Propagates glass effect to all child fields. Called automatically when the parent form uses `asGlass()`.

The `column` parameter (1-indexed) determines the grid column the field starts in. The `colspan` parameter sets how many grid columns the field spans. Both are optional ã when omitted, fields flow naturally in the grid.

## Styling

All visual properties are driven by MD3 design tokens. Override them on `:root` or a scoped parent to restyle this component without touching source.

| CSS variable | Role |
|---|---|
| `--md-sys-color-surface` | Form container background |
| `--md-sys-color-error` | Validation error colour |
| `--md-sys-color-primary` | Focus ring colour for child fields |
| `--md-sys-spacing-*` | Field gaps and layout spacing |

See the [Theming guide](../?path=/docs/system-theming--docs) for token reference and custom theme examples.

### Additional notes

- **Form layout** is built on top of `LayoutBuilder` using a vertical layout with `EXTRA_LARGE` gap between the header, fields grid, error message, and toolbar.
- **Field grid** uses CSS Grid with dynamic column count set by `withFields(columnsAmount)`.
- **Error state** for individual fields renders a 1px outline border outside the input wrapper (`outline-1 outline-offset-1 outline-error`).
- **Global error** is rendered as a small label between the fields and toolbar. It is hidden when the error observable emits an empty string.
- **Caption** is rendered as a large label. **Description** is rendered with smaller typography below the caption. Both are hidden when their observable emits an empty string.
- **Glass effect** applies only to the fields and toolbar, **not** to the form container itself. Fields receive translucent backgrounds with backdrop blur.
- **Toolbar** is always rendered at the bottom of the form. The enabled state of the form propagates to all toolbar buttons.
- Fields in the disabled state apply reduced opacity and prevent user interaction.
